 1. Objective

The objective of this testing project is to verify the core functionality of the SauceDemo web application by performing manual functional testing. The project also aims to practice industry-standard QA documentation and improve software testing skills.

---

 2. Scope

The following modules are included in testing:

- Login
- Products
- Product Details
- Shopping Cart
- Checkout
- Logout

---

 3. Out of Scope

The following areas are not included in this testing cycle:

- Payment gateway integration
- Performance testing
- Security testing
- API testing
- Mobile responsiveness
- Cross-browser compatibility
- Database validation

---

 4. Test Strategy

Testing will be performed manually using exploratory and functional testing techniques.

The focus will be on:

- Positive test scenarios
- Negative test scenarios
- Input validation
- Navigation
- Basic business logic
- UI observations

---

 5. Test Environment

| Item | Details |
|------|---------|
| Device | MacBook Air M2 |
| Operating System | macOS |
| Browser | Safari |
| Application | SauceDemo |
| Internet | Stable Wi-Fi |

---

 6. Entry Criteria

Testing can begin when:

- The application is accessible.
- Test user credentials are available.
- Core application pages load successfully.

---

 7. Exit Criteria

Testing is considered complete when:

- All planned test scenarios have been executed.
- Test execution results have been documented.
- Any identified issues or improvement suggestions have been recorded.

---

 8. Test Deliverables

The following documents will be produced:

- Test Plan
- Test Scenarios
- Test Cases
- Test Execution Report
- Bug Reports
- Improvement Suggestions
- Test Summary

---

 9. Risks

Possible risks include:

- Limited application functionality (demo application)
- No official requirements document available
- Some business rules cannot be fully verified

---

 10. Approval

This project is created for educational and portfolio purposes to demonstrate manual QA skills and documentation practices.
